# Этап 5 — Engage: NextAction по Context (MVP)

## Цель
Сделать рабочий Engage-экран, где пользователь выбирает действия по реальным ограничениям, начиная с контекста.

## Зависимости
- Требует завершения `tasks/4.md`.

## Checklist

### data
- [ ] Убедиться, что у каждого `NextAction` есть обязательный `contextId`.
- [ ] Добавить опциональные поля `timeEstimate` и `energy` в `NextAction` как future-ready (без обязательного UI на первом шаге).
- [ ] Добавить enum `NextActionStatus` (`active`, `done`) если еще не выделен отдельно.
- [ ] Добавить индекс/селектор по `contextId` в состоянии (мемоизированный фильтр).

### state
- [ ] Добавить в `src/hooks/useBoardState.ts` состояние `selectedContextId`.
- [ ] Добавить действие `setSelectedContext`.
- [ ] Добавить селектор `visibleNextActions` по выбранному контексту.
- [ ] Добавить режим `All contexts` (показывать все активные действия).
- [ ] Добавить действие завершения next action (`markNextActionDone`).
- [ ] Добавить оптимистичное обновление счетчиков действий по контекстам.

### ui
- [ ] Создать `src/components/ContextFilterBar.tsx` (или `src/features/engage/ContextFilterBar.tsx`) со списком контекстов.
- [ ] Добавить в UI счетчик активных next actions у каждого контекста.
- [ ] Создать `src/components/NextActionList.tsx` для отображения отфильтрованных действий.
- [ ] Добавить кнопку `Done` на карточке next action.
- [ ] Добавить empty-state для случая, когда в контексте нет активных действий.
- [ ] Добавить переключатель между `Inbox` и `Engage` режимом в `src/App.tsx`.

### validation
- [ ] Проверить, что при смене контекста список действий пересчитывается корректно.
- [ ] Проверить, что `Done` скрывает действие из активного списка.
- [ ] Проверить, что счетчик контекста уменьшается после завершения действия.
- [ ] Проверить, что режим `All contexts` показывает объединенный список.
- [ ] Проверить, что reload сохраняет выбранный контекст и статусы действий.

### manual-test
- [ ] Создать минимум 4 next actions с разными контекстами.
- [ ] Поочередно выбрать каждый контекст и сверить состав списка.
- [ ] Отметить 2 действия как выполненные, убедиться что они не мешают выборке.
- [ ] Проверить сценарий «нет действий в контексте» и корректность empty-state.

## Definition of Done
- Engage-экран показывает `NextAction` по выбранному контексту.
- Контекстный фильтр и счетчики работают стабильно.
- Выполненные действия исключаются из активной выборки.
- Функциональность соответствует MVP-части «NextAction Lists by Context».
