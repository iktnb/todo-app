# Этап 4 — Clarify Wizard (MVP)

## Цель
Внедрить обязательный clarify-процесс для Inbox items по decision tree из `FLOW.md`: actionable? one-step?

## Зависимости
- Требует завершения `tasks/3.md`.

- [ ] Добавить тип `SomedayItem` (минимум `id`, `title`, `notes`, `reviewAt`, `createdAt`).

### ui
- [ ] Добавить кнопку `Clarify` в `src/components/TaskCard.tsx` для raw inbox item.
- [ ] Создать MVP-компонент wizard в `src/components/ClarifyWizard.tsx` (или `src/features/clarify/ClarifyWizard.tsx`).
- [ ] Показать шаг 1: «Это actionable?» (Yes/No).
- [ ] Показать шаг 2 (если Yes): «Это one-step?» (Yes/No).

### validation

- [ ] Проверить, что outcome `someday` создает запись в someday.
- [ ] Проверить, что outcome `trash` реально удаляет запись.

## Definition of Done
- Decision tree из `FLOW.md` реализован в UI и state.
- Все 4 outcome корректно меняют доменное состояние.
