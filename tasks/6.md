# Этап 6 — Project View (MVP)

## Цель
Добавить отдельный Project View и обеспечить GTD-инвариант: у каждого активного проекта есть минимум один NextAction.

## Зависимости
- Требует завершения `tasks/4.md`.
- Желательно завершить `tasks/5.md` для переиспользования списка NextAction.

## Checklist

### data
- [ ] Убедиться, что `Project` хранит `status`, `reviewAt`, `createdAt`.
- [ ] Добавить явную связь `NextAction.projectId` (nullable для несвязанных действий).
- [ ] Добавить селектор `projectActions(projectId)` в state-слое.
- [ ] Добавить селектор `projectsWithoutNextAction`.

### state
- [ ] Добавить CRUD-операции для проектов: create/update status/update title.
- [ ] Добавить действие привязки NextAction к проекту.
- [ ] Добавить действие отвязки NextAction от проекта.
- [ ] Добавить guard: нельзя переводить проект в `active`, если у него 0 next actions.
- [ ] Добавить guard: при удалении последнего next action из активного проекта показывать предупреждение.
- [ ] Добавить авто-вычисление project health (`healthy` / `missing_next_action`).

### ui
- [ ] Создать `src/features/projects/ProjectView.tsx` как отдельный экран/секцию.
- [ ] Создать `src/features/projects/ProjectCard.tsx` с блоком «связанные next actions».
- [ ] Добавить кнопку «Создать проект» в project view.
- [ ] Добавить inline-редактирование названия проекта.
- [ ] Добавить визуальный индикатор проектов без next action.
- [ ] Добавить быстрый переход из project card к созданию next action.

### ui (design-system compliance)
- [ ] Вынести переиспользуемую карточку проекта в `design-system/components/ProjectCard.tsx`.
- [ ] Экспортировать компонент из `design-system/components/index.ts`.
- [ ] Экспортировать компонент из `design-system/index.ts`.
- [ ] Добавить story в `design-system/stories/ProjectCard.stories.tsx` (минимум 3 состояния: healthy, missing_action, done).
- [ ] Переключить `src/features/projects/ProjectView.tsx` на использование компонента из design-system.

### validation
- [ ] Проверить, что новый проект появляется в `active` и виден в Project View.
- [ ] Проверить, что проект без действий подсвечивается как проблема.
- [ ] Проверить, что добавление next action снимает подсветку проблемы.
- [ ] Проверить, что completed-проект не попадает в список активных.
- [ ] Проверить, что reload сохраняет связи Project ↔ NextAction.

### manual-test
- [ ] Создать проект из clarify outcome и вручную из Project View, убедиться что оба сценария работают.
- [ ] Привязать 2 действия к проекту, затем отвязать оба, проверить предупреждение.
- [ ] Закрыть проект со связанными выполненными действиями, убедиться в консистентном статусе.
- [ ] Пройти storybook story для ProjectCard и визуально проверить состояния.

## Definition of Done
- Есть рабочий Project View с управлением проектами.
- Инвариант «active project has next action» технически контролируется.
- Связь Project ↔ NextAction прозрачна в UI.
- Reusable project UI вынесен в design-system и покрыт storybook.
